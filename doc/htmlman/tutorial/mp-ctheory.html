MetaPRL Tutorial: the constructive theory

We have now defined the parts of a constructive logic. We have:

  • a well-formedness judgment type,
  • constants true and false,
  • simple operators implies, and, and or,
  • a derived operator not,
  • a universe of formulas univ,
  • quantifiers all and exists,
  • structural rules.

We have defined display forms for each of these operators, and included tactics for automating their application.

At this point, we can put together a complete constructive first order logic. The process is pretty simple: we simple define a new module that includes each of the operators.

The Fol_ctheory interface

In the interface, we just include each of the module for the operators.

1. Create the constructive interface in the file Fol_ctheory:

include Fol_type
include Fol_true
include Fol_false
include Fol_implies
include Fol_and
include Fol_or
include Fol_not
include Fol_univ
include Fol_all
include Fol_exists

The Fol_ctheory implementation

The implementation includes the same theories as the interface. In addition, we establish display precedence relations that cross the module boundaries.

2. Create the implementation with the include statements of the interface, and the following relations between the precedences in the file fol_ctheory.ml:

open Fol_implies
open Fol_and
open Fol_or
open Fol_not
open Fol_all
open Fol_exists

prec prec_implies < prec_and
prec prec_implies < prec_or
prec prec_or < prec_and
prec prec_and < prec_not
prec prec_all < prec_implies
prec prec_exists < prec_implies

At this point, we have a complete theory. Before going on to the classical theory, let's prove a simple theorem.

3. Create a theorem object:

interactive or_choice 'H :
   sequent ['ext] { 'H >- "type"{'A} } -->
   sequent ['ext] { 'H >- "type"{'B} } -->
   sequent ['ext] { 'H >- "type"{'C} } -->
   sequent ['ext] { 'H >- (('A or 'B) => 'C) => 'A => 'C }
