Download MetaPRL

MetaPRL can be downloaded from ftp://ftp.cs.cornell.edu/pub/nuprl/MetaPRL/. You will need four packages:

  1. Objective Caml: our version contains a small native code profiler patch.
  2. CamlP4: our version contains a small patch to compile the CamlP4 distribution in native code, native code profiler patch and a patch to install interface files needed by MetaPRL to evaluate programs in the toploop
  3. Ensemble: you need Ensemble only if you want to use the distributed prover.
  4. MetaPRL

All source modules are available as source, and RedHat 5.x RPM packages files for Linux x86 at ftp://ftp.cs.cornell.edu/pub/nuprl/MetaPRL/RPM/. There is also a Win32 distribution that we'll be placing online soon (if you don't see it contact Jason Hickey). You will need the Cygnus Systems GNU-Win32 distribution.

If you are going to build OCaml and/or Camlp4 yourself, you can get our patches at ftp://ftp.cs.cornell.edu/pub/nuprl/MetaPRL/patches/

CVS sources

You can browse the MetaPRL CVS repository.

If you plan any development in MetaPRL, you may consider using the public CVS repository. This will allow you to have access to the latest changes in the distribution. The normal service is read-only, but if you would like to become a contributor with write access, contact Jason Hickey or Alexey Nogin.

The CVS server is on the machine ensemble01.cs.cornell.edu. First log into the server by running 

% cvs -d :pserver:anoncvs@ensemble01.cs.cornell.edu:/cvsroot login
The password is anoncvs. It will be stored on your computer, so you only have to run the login command once.

Now you can checkout the distribution by running:

% cvs -d :pserver:anoncvs@ensemble01.cs.cornell.edu:/cvsroot checkout MetaPRL
You can find more information on CVS at Cyclic Software's pages

Installation and configuration

If your are not using the provided RPMs, then in order to install OCaml and CamlP4 download and unpack the distributions, than apply the patches. After that proceed as described in the INSTALL files in the distributions.

To install Ensemble, unpack the distrubution, and edit the <ensemble>/mk/config.mk file. You will want to use the Ensemble socket library (not the Unix socket implementation), then change directories to <ensemble>/def, and run make. You can also build the native-code libraries by running make in the <ensemble>/opt directory. More information on Ensemble is available at the Ensemble Home Page.

You may need to set your environment variables CAMLLIB and CAMLP4LIB to point to the directories where you have installed OCaml and Camlp4 files. The defaults are /usr/lib/ocaml and /usr/lib/camlp4 respectively. If you are using Ensemble, set ENSROOT=<root of the ensemble source tree> Also, go to the OCaml source tree and copy the folowing files:

  • parsing/location.cmi
  • parsing/location.mli
  • parsing/longident.cmi
  • parsing/longident.mli
  • parsing/parsetree.cmi
  • parsing/parsetree.mli
  • typing/typecore.cmi
  • typing/typecore.mli
into the $CAMLLIB directory

To install MetaPRL, unpack the distribution into some directory <meta-prl> and build the distribution with these steps:

% cd <meta-prl>
% make all   (builds bytecode)
or
% make opt   (builds native code)

The MetaPRL programs are <meta-prl>/editor/ml/mptop (bytecode) and <meta-prl>/editor/ml/mpopt (native code). You will need to install the "MetaPRL font" to be able to interact with some sense. The MetaPRL font is in <meta-prl>/editor/fonts/mp12.bdf. You'll get the best success if you use MetaPRL from a shell window within Emacs. You can start Emacs with the MetaPRL font by running emacs -fn mp12.

That's it. To navigate the MetaPRL logics and programs, you will need to read the User Guide.
